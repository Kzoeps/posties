# AT Protocol Lexicons

This app stores sticky quote data as custom AT Protocol records in each user's DID-owned PDS repo. Shareable URLs use the user's handle, but handle values are resolved to DIDs before any quote/settings records are read or written. The NSID authority is `kzoeps.com`, represented as `com.kzoeps`.

All records described here are public PDS data. Users should not store private quotes, secrets, access tokens, API keys, or sensitive personal notes unless a future encrypted storage design is added.

## Collections

| Purpose | Collection NSID | Record key |
| --- | --- | --- |
| Sticky quote records | `com.kzoeps.stickyquotes.canvas.quote` | `tid` generated by the PDS |
| Canvas settings | `com.kzoeps.stickyquotes.canvas.settings` | literal `self` |

Source lexicon JSON files live in `src/lib/atproto/lexicons/`:

- `src/lib/atproto/lexicons/com.kzoeps.stickyquotes.canvas.quote.json`
- `src/lib/atproto/lexicons/com.kzoeps.stickyquotes.canvas.settings.json`

The NSIDs must stay in sync with `src/lib/atproto/nsids.ts` and `src/lib/config.ts`.

## Where records live

Quote records are stored in the active user's repo at AT URIs shaped like:

```text
at://<did>/com.kzoeps.stickyquotes.canvas.quote/<rkey>
```

The singleton settings record is stored at:

```text
at://<did>/com.kzoeps.stickyquotes.canvas.settings/self
```

Public handle-page read flow:

1. Normalize and validate the `/:handle` route segment.
2. Resolve the handle to a DID with `com.atproto.identity.resolveHandle`.
3. Resolve that DID document to the current canonical `at://<handle>` alias and `#atproto_pds` service endpoint.
4. Redirect to the canonical handle when the route handle differs.
5. List all records in `com.kzoeps.stickyquotes.canvas.quote` from that DID's repo using the resolved PDS endpoint. No OAuth session is required for this read.
6. Validate every returned record locally.
7. Normalize `uri`, `cid`, parsed `rkey`, and record fields into app quote objects.

Signed-in owner write flow still uses OAuth and compares the active OAuth DID with the page owner DID before rendering write controls.

Write flow:

- Writes are owner-only in the UI: Add/Edit/Delete/retry controls and persisted moves are available only when `activeDid === ownerDid`; non-owners can drag notes locally without PDS writes.
- Create uses `com.atproto.repo.createRecord` and lets the PDS generate a `tid` rkey.
- Edit and move use `com.atproto.repo.putRecord` with the latest known `cid` as `swapRecord` when available.
- Delete uses `com.atproto.repo.deleteRecord` with `swapRecord` when available.
- Settings use the literal rkey `self`.

## `com.kzoeps.stickyquotes.canvas.quote`

One record represents one sticky note on the owner's infinite canvas.

| Field | Required | Type/constraints | Description |
| --- | --- | --- | --- |
| `$type` | Yes | constant `com.kzoeps.stickyquotes.canvas.quote` | AT Protocol record type discriminator. |
| `schemaVersion` | Yes | integer constant `1` | Enables future migrations while keeping v1 records explicit. |
| `text` | Yes | string, 1-2000 graphemes | Plain quote text rendered on the sticky note. Never render as HTML. |
| `author` | No | string, max 200 graphemes | Optional person or source credited for the quote. |
| `sourceTitle` | No | string, max 300 graphemes | Optional title of the book, article, talk, or other source. |
| `sourceUri` | No | URI string, max 2048 bytes | Optional absolute URI for the source. |
| `position.x` | Yes | integer, `-1000000000` to `1000000000` | Sticky note top-left x-coordinate in canvas world units. |
| `position.y` | Yes | integer, `-1000000000` to `1000000000` | Sticky note top-left y-coordinate in canvas world units. |
| `size.width` | No | integer, 160-600 | Optional width in world units. The app default is 240. |
| `size.height` | No | integer, 120-500 | Optional height in world units. The app default is 180. |
| `rotationDegX100` | Yes | integer, -800 to 800 | Persistent note rotation in hundredths of a degree. |
| `color` | Yes | enum: `yellow`, `pink`, `blue`, `green`, `purple`, `orange`, `gray` | Fixed palette token used by all app clients. |
| `zIndex` | No | integer, 0-1000000 | Optional stacking order for overlapping notes. |
| `createdAt` | Yes | datetime string | Time the quote record was created. |
| `updatedAt` | Yes | datetime string | Time the quote record was last changed. |

### Coordinates

Coordinates are integer world coordinates, not screen pixels. Panning and zooming the viewport must not change stored quote positions. During drag, UI code can use floating-point math for smooth movement, but final persisted `x` and `y` values must be rounded to integers before writing to the PDS. Viewer-only drags are accepted locally in the browser and are not written back.

The v1 placement algorithm ignores rotation for collision checks and uses axis-aligned bounds plus a gap. That keeps placement deterministic and avoids tying record validation to browser-specific transform math.

### Rotation

`rotationDegX100` stores fixed-point degrees:

```text
rendered degrees = rotationDegX100 / 100
```

Examples:

- `-375` renders as `-3.75deg`
- `0` renders as `0deg`
- `425` renders as `4.25deg`

Generate this value once when creating a note and persist it. Do not generate random rotation during render; otherwise notes would visibly change tilt after reload.

## `com.kzoeps.stickyquotes.canvas.settings`

One record represents optional per-user canvas preferences. The record key is always `self`.

Do not store quote IDs or handles in this record. Quote records are discoverable by listing the quote collection, and duplicating note IDs or mutable handles here would create sync and conflict problems.

| Field | Required | Type/constraints | Description |
| --- | --- | --- | --- |
| `$type` | Yes | constant `com.kzoeps.stickyquotes.canvas.settings` | AT Protocol record type discriminator. |
| `schemaVersion` | Yes | integer constant `1` | Enables future settings migrations. |
| `defaultColor` | No | enum: `yellow`, `pink`, `blue`, `green`, `purple`, `orange`, `gray` | Optional default color for newly created notes. |
| `lastViewport.x` | No, unless `lastViewport` exists | integer, `-1000000000` to `1000000000` | Last viewport center x-coordinate in world units. |
| `lastViewport.y` | No, unless `lastViewport` exists | integer, `-1000000000` to `1000000000` | Last viewport center y-coordinate in world units. |
| `lastViewport.zoomX1000` | No, unless `lastViewport` exists | integer, 200-3000 | Last zoom multiplier as `zoom * 1000`. |
| `updatedAt` | Yes | datetime string | Time the settings record was last changed. |

Current implementation note: settings read/write helpers and tests exist, but the UI does not yet restore the viewport from `lastViewport`.

### Zoom

Settings store zoom as the fixed-point integer `zoomX1000`:

```text
rendered zoom = zoomX1000 / 1000
```

Examples:

- `200` renders as `0.2x`
- `1000` renders as `1.0x`
- `3000` renders as `3.0x`

This mirrors the quote rotation strategy: persisted records use integers so validation is deterministic across clients.

## Public-data policy

PDS records in these collections are public or discoverable by normal AT Protocol repository tooling. V1 intentionally does not encrypt quote text, authors, source metadata, positions, colors, or settings.

Implementation implications:

- Show a public-data warning before users create quotes and when viewing a public handle page.
- Store user-entered quote text as plain text only.
- Never render quote text or metadata with unsafe HTML APIs.
- Do not put OAuth tokens, API keys, handles, or other mutable/account-secret data in these records.

## Local validation expectations

The PDS may not enforce every custom lexicon constraint uniformly. App code must validate locally before writing and after reading records. Validators live in `src/lib/atproto/validation.ts` and return errors that name the bad field, what went wrong, and how to fix it.

PDS write helpers default to relaxed custom lexicon validation because app-side validation is mandatory and PDS support can vary. If a future deployment confirms strict validation works everywhere required, this default can be revisited.

## Migration policy

Both records include `schemaVersion: 1`.

For backward-compatible additions:

1. Add optional fields only.
2. Keep existing field meanings stable.
3. Update local validators and docs with defaults for missing optional fields.

For breaking changes:

1. Create a migration plan before shipping clients that write the new shape.
2. Prefer a new optional field plus app-side migration over changing the meaning of an existing field.
3. If a new collection is required, choose a new NSID under `com.kzoeps.stickyquotes.*` and keep read support for v1 records until users are migrated.
4. Never silently rewrite public PDS records without a user-visible reason and an error recovery path.
